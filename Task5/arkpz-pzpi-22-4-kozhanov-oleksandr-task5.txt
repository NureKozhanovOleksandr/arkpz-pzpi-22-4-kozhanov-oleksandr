1 МЕТА

Метою роботи є розгорнути програмну систему для контролю стану тварин у клініках та продемонструвати її роботу.

2 ХІД РОБОТИ

Протягом виконання лабораторної роботи програмну систему для контролю стану тварин в клініках було розгорнуто з використанням Node.js та Wokwi для емуляції IoT пристрою. 
Для встановлення системи необхідно завантажити архів з усіма компонентами та виконати налаштування, описані у відео-звіті (див. Додаток А).
При розгортанні системи було зроблено наступні припущення:
–	користувач установив Node.js та MongoDB;
–	на комп'ютері користувача встановлено ngrok для створення тунелю;
–	у користувача є доступ до симулятора Wokwi для роботи з розумним пристроєм.
Через особливості роботи симулятора Wokwi, підключення до локальної мережі здійснюється через ngrok тунель. У реальному проекті передбачається пряме підключення пристрою через локальну Wi-Fi мережу.
Загальний опис інженерних рішень:
а) серверна частина: 
1)	мова програмування: JavaScript (Node.js);
2)	фреймворк: Express.js;
3)	база даних: MongoDB;
4)	REST API для взаємодії з клієнтами;
б) розумний пристрій:
1)	платформа: ESP32;
2)	компоненти: датчик температури DHT22;
3)	середовище розробки та емуляції: Wokwi;
4)	спосіб зв'язку з сервером: HTTP через ngrok тунель;
в) розгортання системи:
1)	витягання файлів з архіву;
2)	встановлення залежностей через npm;
3)	налаштування змінних середовища;
4)	запуск через run.bat.
